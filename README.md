# Analysis 数据分析系统

对清洗后的充电数据做标准化产品输出（表格与图表可下载），并预留一键入库界面。侧栏：**入库**、**标准化数据产品-由数据库生成**（导入/库表）、**标准化数据产品-由原始表生成标准00表**（能源局类月度汇总 xlsx → 标准 00 表，说明见 `docs/由原始表生成标准00表_功能说明.md`）、**标准化数据产品-由标准00表生成**（00 表转七表）、**功率表添加环比**（多月份功率段 xlsx 目录内原地写环比，规则与 00 表转化一致）。

## 功能

### 一键入库（页面上方）

- 展示数据库配置（与 charging-agent 的 config 一致，环境变量可覆盖）。支持 **MySQL** 与 **PostgreSQL**（PostgreSQL 须先选 Schema）。
- **入库方式**三选一：**已有表追加数据**、**新增表（空白）**（按上传文件列与类型建表）、**新增表（复制其他表结构）**（从当前库/Schema 的下拉列表选一张表，`CREATE TABLE ... LIKE` 复制结构后再导入）。
- 充电桩表/充电站表入库时：先将数据**全部写入与目标表同结构的临时表**（每次导入前 `TRUNCATE` 临时表）；若不能完整写入临时表则报错并中止，**不会写入正式表**。**PostgreSQL** 下写入临时表使用 **`COPY FROM STDIN`（CSV）**，速度与 psql `\\copy` 同量级；MySQL 仍为 `pandas.to_sql` 批量插入。校验通过后，在**单事务**中从临时表 `INSERT` 到正式表；若发生主键/唯一约束冲突等错误则**回滚**本次对正式表的插入。若目标表含 **uid** 列，写入正式表前会按 uid 去重（PostgreSQL 使用 `DISTINCT ON (uid)` 并按「入库时间」降序；MySQL 8+ 使用 `ROW_NUMBER()`）。
- **新增表（空白/复制结构）** 且 **PostgreSQL** 时，可勾选 **「快速入库」**：跳过临时表与去重，**直接 `COPY` 到目标表**（适合新建空表整表导入；不做 uid 去重）。
- 选择清洗后文件后点击「执行入库」；完成后展示成功/失败摘要。CSV 缓冲格式自测见 `tests/test_pg_copy_csv_format.py`。

### 标准化数据产品-由数据库生成

- **数据来源**二选一：**导入文件**（上传 Excel/CSV）或 **从数据库选择表**（从 evdata 库拉取表列表并加载为 DataFrame）。
- 无论哪种来源，均需在「请确认表类型」下拉框中选择 **充电桩表** 或 **充电站表**，以用户选择为准参与后续所有产品计算。
- 生成与开放服务平台一致的数据表（排行榜、各运营商、全国/各省、城市群、功率段、高速、车桩比），支持导出 Excel/CSV；功率段支持导出图表 PNG。**单独导出「功率段分布」xlsx** 时，最前增加 Sheet「总」（表头：省份、功率段、数量、占比、环比），其后仍为各省 Sheet；**一键七表 ZIP** 内功率段文件结构不变。

## 数据产品与表类型、必备列

| 产品 | 充电桩表 | 充电站表 | 必备列（桩） | 必备列（站） |
|------|----------|----------|--------------|--------------|
| 存量市场份额榜 | 支持 | 支持 | 运营商名称/上报机构、充电桩编号 | 运营商名称、所属充电站编号 |
| 城市榜 Top10 | 支持 | 需城市列 | 城市或城市_中文、充电桩编号 | 城市或城市_中文 |
| 各运营商数据 | 支持 | 支持 | 同上 | 同上 |
| 全国数据-汇总卡片 | 支持 | 支持 | 充电桩编号 | 所属充电站编号 |
| 全国数据-各省排名 | 支持 | 需省份列 | 省份或省份_中文、充电桩编号 | 省份或省份_中文 |
| 各省数据-单省概览 | 支持 | 需省份列 | 同上 + 额定功率等 | 同上 + 站点总装机功率等 |
| 核心城市群 | 支持 | 需省份列 | 省份、充电桩编号 | 省份、所属充电站编号 |
| 功率段分布 | 支持 | 支持 | 额定功率 | 站点总装机功率 |
| 高速公路建设 | 占位 | 占位 | — | — |
| 车桩比 | 占位 | 占位 | — | — |

充电站表若缺少「省份」「城市」列，各省排名、城市榜、各省数据、核心城市群不展示并给出提示。功率段分布：桩表按**额定功率**分档，站表按**站点总装机功率**分档，界面与导出中会注明。

## 环境与运行

```bash
pip install -r requirements.txt
streamlit run app.py
```

Windows 可双击 `启动_Analysis.bat` 启动。从数据库选表需配置 `config.py` 中的 DB 并确保 evdata 库可访问（依赖 `pymysql`）。选用 **PostgreSQL** 时，界面默认库名为 **evdata**、默认密码为 **Admin2026**（Host/Port/User 仍为 localhost / 5432 / postgres，可在页面修改）。

## 项目结构

- `app.py`：Streamlit 入口，侧栏切换入库与两类标准化数据产品生成。
- `config.py`：数据库配置（环境变量可覆盖）。
- `db_helper.py`：拉取 evdata 表列表、读取指定表为 DataFrame。
- `handlers/`：各板块聚合与表格生成（排行榜、运营商、全国、省份、城市群、功率段、高速、车桩比）。

数据格式与 merge 系统输出一致，参见 merge_app 的《数据格式与入库规范说明》。
