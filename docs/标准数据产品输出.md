# 标准数据产品输出

本文档罗列「输出标准化产品」页面需要的表/图（产品），每个产品对应【数据来源】、数据来源表中的【具体字段】、如何实现及是否【涉及跨数据表】。便于逐个精确调试与后续扩展。

**数据来源约定**：页面当前仅支持**单一数据源**——用户上传或从库中选择**一张**表，并指定为「充电桩表」或「充电站表」。评估数据来源时**优先站表**，且选定的表必须能提供该产品所需的**全部具体字段**，否则选用另一张表。**是否满足条件**指在当前数据来源表具备上述具体字段的前提下，现有数据能否生成该表（否则需在备注中说明缺乏什么）。**涉及跨数据表**指：该产品所需原始数据是否不仅在一张表里、是否需要连接两张及以上表进行查询；若仅用当前选中的这一张表则填「否」。

**统计口径约定**：后续统计**充电站数量**统一用【充电站内部编号】去重计数；**充电桩数量**统一用【序号】计数。充电站统计时**仅统计站内充电桩数量 > 2 的充电站**（站内桩数 ≤ 2 的不计入）：若数据来自充电站表，可用「站点内桩总数 > 2」过滤后再对充电站内部编号去重；若数据来自充电桩表，则先按充电站内部编号（或所属充电站编号）分组计桩数，剔除站内桩数 ≤ 2 的站，再对剩余站的充电站内部编号去重计数。

**表名约定**：下文 SQL 基于清洗后的两张表——充电桩表 `charge`、充电站表 `station`，字段与入库表头一致。

### P1 排行榜

#### 1.1 市场份额榜

- **所需原始表**：charge（充电桩表）。
- **字段需求**：`运营商名称`；count(`序号`)。
- **实现方式**：按运营商名称分组统计充电桩数量（序号计数），排序取 Top10，计算各运营商设施总量及全国占比（市场份额）。

- **备注**：—。

```sql
SELECT `运营商名称`, COUNT(`序号`) AS total
FROM charge
WHERE `运营商名称` IS NOT NULL
GROUP BY `运营商名称`
ORDER BY total DESC
LIMIT 10;
```

#### 2.10 充电站

- **所需原始表**：station，charge（站内桩数从 station.站点内桩总数 或 charge 分组计）。
- **字段需求**：`充电站内部编号`；`站点内桩总数`（过滤用，>2 才计入）。
- **实现方式**：仅统计站内桩数>2 的充电站；按运营商对充电站内部编号去重计数。
- **是否满足条件**：是。
- **备注**：包含 >2 桩过滤逻辑。

- **查询 SQL（station）**：

```sql
SELECT `运营商名称`, COUNT(DISTINCT `充电站内部编号`) AS 充电站数
FROM station s
WHERE (
  CASE
    WHEN s.`站点内桩总数` IS NOT NULL THEN s.`站点内桩总数`
    ELSE (SELECT COUNT(*) FROM charge c WHERE c.`充电站内部编号` = s.`充电站内部编号`)
  END
) > 2
GROUP BY `运营商名称`
ORDER BY 充电站数 DESC;
```

#### 3.2 充电站总数

- **所需原始表**：station，charge（若 station 无省份则需关联 charge 取省份）。
- **字段需求**：`省份_中文`；`充电站内部编号`；站内桩数>2 过滤。
- **实现方式**：仅统计站内桩数>2 的站；按省份对充电站内部编号去重计数。若 station 无省份列，则用 charge 按充电站内部编号取省份后汇总。
- **是否满足条件**：是。
- **备注**：—。

- **查询 SQL（station，假设 station 有省份_中文列时）**：

```sql
SELECT `省份_中文`, COUNT(DISTINCT `充电站内部编号`) AS 充电站数
FROM station s
WHERE (
  CASE WHEN s.`站点内桩总数` IS NOT NULL THEN s.`站点内桩总数`
  ELSE (SELECT COUNT(*) FROM charge c WHERE c.`充电站内部编号` = s.`充电站内部编号`) END
) > 2
GROUP BY `省份_中文`
```

### P6 功率段分布

#### 6.1 功率段分布

- **所需原始表**：charge。
- **字段需求**：`额定功率`（分档：如 <120、120–250、250–480、480–960、≥960 kW）。
- **实现方式**：将额定功率分档，按档统计充电桩数量及占比；可同时输出表+柱状图。
- **是否满足条件**：是。
- **备注**：—。

- **查询 SQL（charge）**：

```sql
SELECT
  CASE
    WHEN `额定功率` < 120 THEN 'p<120kW'
    WHEN `额定功率` < 250 THEN '120-250kW'
    WHEN `额定功率` < 480 THEN '250-480kW'

FROM charge
WHERE `额定功率` IS NOT NULL
GROUP BY 1
ORDER BY FIELD(1, 'p<120kW', '120-250kW', '250-480kW', '480-960kW', '≥960kW');
```

## 三、按数据表类型汇总：必备字段



数据来源按**优先站表**评定，且选定表必须能提供该产品所需的全部具体字段；否则选用充电桩表。**是否满足条件**表示在现有表结构下能否生成该表，否时在备注中说明缺乏字段。

- **充电桩表**：  
充电桩数量用 `序号` 计数；充电站数量用 `充电站内部编号` 去重，且仅统计站内桩数 > 2 的站（先按充电站内部编号分组计桩数再过滤）；运营商维度依赖 `运营商名称` 或 `上报机构`；地域维度依赖 `省份_中文`/`省份`、`城市_中文`/`城市`；功率段依赖 `额定功率`；型号榜依赖 `充电桩型号`。
- **充电站表**：  
充电站数量用 `充电站内部编号` 去重，且仅统计 `站点内桩总数` > 2 的站；运营商依赖 `运营商名称` 或 `上报机构`；功率段依赖 `站点总装机功率`。标准表无 `省份`、`城市`，故需省份/城市维度的产品用充电桩表。

## 四、涉及跨数据表说明

当前所有产品均**不涉及跨数据表**：仅基于用户选择的**一张**充电桩表或充电站表计算。若后续出现以下需求，则需扩展为跨表或外部数据：

- **环比增量/环比增速**：需历史期同一表或历史汇总表。
- **车桩比**：需新能源车保有量等车辆数据（另一张表或外部接口）。
- **高速公路建设**：若用独立高速台账，则需关联该表。

实现与调试时，按总览表所选**数据来源**表加载数据，保证该表具备该产品「具体字段」所列字段；若「是否满足条件」为否，则当前仅能占位或近似输出，备注中已说明所缺字段。
