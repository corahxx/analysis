# Analysis 布局与数据源改造计划

## 一、页面布局调整

- **当前**：侧栏切换「输出标准化产品」「一键入库」，两功能分页展示。
- **目标**：单页上下结构，**入库在上、功能2 在下**。
  - 页面上半部分：一键入库（数据库配置 + 入库方式 + 执行与摘要）。
  - 页面下半部分：输出标准化产品（数据源选择 + 表类型确认 + 产品列表与导出）。

实现方式：在 [app.py](d:\文件\充电代码工作\analysis\app.py) 中去掉侧栏功能切换，改为从上到下依次渲染「入库」区块和「输出标准化产品」区块；保留侧栏仅作说明或留空。

## 二、入库功能增强

### 2.1 入库方式二选一

- **evdata 表追加数据**：将本次数据追加到已存在的 evdata 库表中（如固定表名：充电桩表、充电站表，或与用户选择的类型对应）。
- **新增表导入数据**：将本次数据导入为新表（表名可由用户输入或按日期/类型自动生成）。

UI：在「执行入库」前增加单选框或下拉框，选项为「evdata 表追加数据」「新增表导入数据」。若选新增表，可增加表名输入框（可选）。

### 2.2 执行与摘要

- 保留「选择清洗后文件」与「执行入库」按钮；入库逻辑未实现时仍可为占位。
- 入库结束后输出摘要：**成功条数**、**失败条数**（及可选：失败原因摘要、失败样例行）。与 plan 中「成功多少条，失败多少条」一致。

## 三、功能2 数据来源与表类型

### 3.1 两种数据获得方式

**方式一：导入**

- 用户上传 Excel/CSV（清洗后的表）。
- **必须**通过下拉框选择表类型：「充电桩表」或「充电站表」。不再仅靠自动检测；用户选择为准，用于后续所有产品生成。

**方式二：数据库表选择**

- 连接 config 中的 DB（evdata），列出库内表名（如 `SHOW TABLES` 或读取元数据）。
- 用户从下拉框选择一张表，加载为 DataFrame。
- 选表后**必须**再通过下拉框确认「表类型」：「充电桩表」或「充电站表」。因库表名未必能区分桩/站，需用户确认。

两种方式下，**表类型**均以用户选择为准，并传入各 handler 的 `for_pile`（True=充电桩表，False=充电站表）。

### 3.2 数据源 + 表类型 的 UI 顺序

1. 选择数据来源：单选「导入文件」或「从数据库选择表」。
2. 若导入：显示文件上传组件；上传成功后，显示「请确认表类型」下拉框（充电桩表 / 充电站表），默认可依当前自动检测结果高亮。
3. 若数据库：显示数据库表下拉框（需先连接 DB 取表列表）；选择表后加载数据，再显示「请确认表类型」下拉框（充电桩表 / 充电站表）。
4. 确认表类型后，下方展示「输出标准化产品」各板块（与现有一致），所有计算均基于当前 DataFrame 与所选表类型。

## 四、基于 merge 两种表的数据产品生成规则

依据 [数据格式与入库规范说明.md](D:\文件\充电代码工作\merge\merge_app\数据格式与入库规范说明.md) 与 merge 的 `STANDARD_COLUMNS` / `STATION_STANDARD_COLUMNS`，约定如下。

### 4.1 充电桩表（清洗后）可用字段与口径

- **主键/统计口径**：设施量 = `充电桩编号.nunique()`；行数仅当无编号时退化为 `len(df)`。
- **运营商**：列优先 `运营商名称`，否则 `上报机构`。
- **地域**：省优先 `省份_中文` 否则 `省份`；城市优先 `城市_中文` 否则 `城市`。
- **功率**：`额定功率`（已清洗为 kW），用于功率段分布。
- **型号/厂商**：`充电桩型号`、`充电桩生产厂商名称`，用于型号榜等（若有需求）。

### 4.2 充电站表（清洗后）可用字段与口径

- **主键/统计口径**：设施量 = 站数 = `所属充电站编号.nunique()` 或 `len(df)`（一行一站）。
- **运营商**：`运营商名称`（或上报机构若存在）。
- **地域**：充电站标准列含 **区县**，不含「省份」「城市」。若库表或后续 merge 增加省份/城市列，则与桩表一致用 `省份_中文`/`省份`、`城市_中文`/`城市`；**当前仅区县时**：
  - **各省排名 / 全国数据-省份维度**：若无省份列，该板块不生成或提示「充电站表无省份列，无法生成各省排名」。
  - **城市榜**：若无城市列，不生成或提示「充电站表无城市列，无法生成城市榜」。
  - 可选：用「区县」做「各区县排名」作为替代产品，与产品命名区分开。
- **功率**：`站点总装机功率`、`交流桩总装机功率`、`直流桩总装机功率`（已 kW），可做「功率段分布」按站点总装机功率分档；与桩表按「额定功率」分档含义不同，需在界面或导出中注明「按站点总装机功率」。

### 4.3 各数据产品与表类型、必备列


| 产品        | 充电桩表     | 充电站表 | 必备列（桩）          | 必备列（站）        | 说明                     |
| --------- | -------- | ---- | --------------- | ------------- | ---------------------- |
| 存量市场份额榜   | 支持       | 支持   | 运营商名/上报机构、充电桩编号 | 运营商名称、所属充电站编号 | 设施量=桩数/站数，占比=单运营商/全国   |
| 城市榜 Top10 | 支持       | 需城市列 | 城市或城市_中文、充电桩编号  | 城市或城市_中文      | 站表无城市则跳过并提示            |
| 各运营商数据    | 支持       | 支持   | 同上              | 同上            | 同排行榜口径                 |
| 全国数据-汇总卡片 | 支持       | 支持   | 充电桩编号           | 所属充电站编号       | 总量=桩数/站数               |
| 全国数据-各省排名 | 支持       | 需省份列 | 省份或省份_中文、充电桩编号  | 省份或省份_中文      | 站表无省份则跳过并提示            |
| 各省数据-单省概览 | 支持       | 需省份列 | 同上 + 额定功率等      | 同上 + 站点总装机功率等 | 按省筛选后聚合                |
| 核心城市群     | 支持       | 需省份列 | 省份、充电桩编号        | 省份、所属充电站编号    | 按规划内省份筛选               |
| 功率段分布     | 支持       | 支持   | 额定功率            | 站点总装机功率       | 桩按单桩功率分档，站按站总功率分档；标注清楚 |
| 高速公路建设    | 占位/需业务字段 | 占位   | —               | —             | 当前无专用字段，保持占位或按省汇总站数    |
| 车桩比       | 占位       | 占位   | —               | —             | 需车辆数据，保持占位             |


### 4.4 Handler 严谨约定

- 每个 handler 在入口处根据 `for_pile` 和 `df.columns` 判断必备列是否存在；若不存在，返回空 DataFrame 或明确标注「缺少 xx 列，无法生成」，不在内部用未定义列。
- 充电站表：`_province_col` / `_city_col` 若取不到，则对应「各省排名」「城市榜」等不生成，并在 app 层提示用户（如「当前为充电站表且无省份列，各省数据不展示」）。
- 功率段：桩表用 [power_handler](d:\文件\充电代码工作\analysis\handlers\power_handler.py) 现有 `额定功率` 分档；站表新增或同一 handler 内分支：按 `站点总装机功率` 分档，图与表标题注明「按站点总装机功率」。

## 五、不确定点与需讨论处

1. **充电站表无省份/城市**：是否允许 merge 或入库阶段为充电站表补充省份/城市（如通过区县映射）？若不允许，则 analysis 对站表只展示「有省份/城市列时才有的板块」，并在 UI 说明。
2. **入库表名**：evdata 追加时，目标表名是否固定为如 `pile_cleaned` / `station_cleaned`，还是由用户在界面上选择？新增表导入时表名规则（例如 `pile_YYYYMMDD`）是否需您指定？
3. **数据库选表**：从 evdata 拉取表列表时，是否过滤只显示「疑似清洗后的表」（如通过表名或列名包含充电桩编号/所属充电站编号）还是显示库内全部表？

## 六、实施顺序建议

1. **app.py 布局**：改为单页，入库区块在上、功能2 在下；移除侧栏切换。
2. **入库 UI**：增加「evdata 表追加」/「新增表导入」选项；保留执行与摘要占位（成功/失败条数）。
3. **功能2 数据源**：增加「导入」与「从数据库选择表」两种入口；两种入口下均增加「表类型」下拉框（充电桩表/充电站表），以用户选择覆盖自动检测。
4. **数据库连接**：在 config 基础上增加「拉取 evdata 表列表」与「读取指定表为 DataFrame」的简单模块（可独立函数或小 handler），供 app 调用；若未实现真实入库，可先仅实现「读表」。
5. **handlers 严谨化**：按上表补充必备列判断；站表无省份/城市时各省排名与城市榜返回空并注明原因；功率段对站表按站点总装机功率分档并标注。
6. **文档**：在 README 或项目内 doc 中写明「数据产品与表类型、必备列」对应关系，便于后续维护与与你讨论。

按上述实施后，入库在上、功能2 在下，入库可选追加或新增表，功能2 支持导入与库表两种数据源且均需确认表类型，各数据表生成规则与 merge 两种清洗表形式一致且可讨论扩展。
