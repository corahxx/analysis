## 三、与 db_helper 的对应关系

- 当前 analysis 建表逻辑见 [db_helper.create_table_from_df](../db_helper.py)：根据上传 DataFrame 的列名与 dtype 推断 MySQL 类型并生成 CREATE TABLE，**不为主键列（如序号）添加 PRIMARY KEY**。
- 经度、纬度在 db_helper 中已按本规范使用 **INT**；其余列类型推断规则与上表一致（日期类 VARCHAR(50)、充电站位置 VARCHAR(600)、功率/电压/电流 DOUBLE、整数计数 BIGINT、默认 VARCHAR(255)/VARCHAR(500)）。
- 若需严格按本表逐列建表，可在 db_helper 中为充电桩表/充电站表维护「列名 → MySQL 类型」映射后优先使用。
