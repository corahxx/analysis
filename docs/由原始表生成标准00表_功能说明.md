# 由原始表生成标准00表 — 功能说明

本文档描述侧栏 **「标准化数据产品-由原始表生成标准00表」** 能力：上传能源局等 **月度原始汇总工作簿**（与 `D:\充电数据汇总\00表合集` 下《00、充换电数据梳理图表-*》类文件一致），自动拼装为与 **`00表标准化-系统输入-YYMM.xlsx`** 相同 **Sheet 名与列顺序** 的标准 00 表，供 **「标准化数据产品-由标准00表生成」** 直接上传转七表。

**原始簿无固定 Sheet 命名**：实现 **不写死** 单一表名，而是对 Sheet 名做归一化后按 **关键词命中 + 排除词** 在多期样例上归纳规则（见 `handlers/raw_to_standard00.py` 中 `resolve_raw_workbook_sheets` / `RawSheetPlan`）。同一逻辑表在不同月份可出现尾部的 `!` / `！`、`-能源局` / `--能源局`、书名号/括号等差异。

**与现有功能关系**：

- **不替代**「由数据库生成」明细清洗链路；原始簿为 **汇总表**，与 merge 明细字段不同。
- **输出**即标准 00 表输入物，与 `docs/由标准00表转化_功能说明.md` 中的 **输入约定**对齐（Sheet：省份、运营商、城市、型号、车企、换电站-运营商）。

---

## 一、输入约定

### 1.1 文件名与统计期

- 文件名中须包含 **六位年月**：`YYYYMM`（如 `202602` 表示 2026 年 2 月）。实现用正则 `(20\d{2})(0[1-9]|1[0-2])` 抽取。
- **输出建议文件名**：`00表标准化-系统输入-{YYMM}.xlsx`，其中 **YYMM** 为 `YYYYMM` 的后四位（例：`202602` → `2602`）。
- 若文件名中 **无法解析** 年月：仍生成工作簿，后缀为 **`YYMM` 字面**（`00表标准化-系统输入-YYMM.xlsx`），请手工改名或修正上传文件名。
- **Streamlit 临时文件**：解析以 **原始上传文件名** 为准（`build_standard00_workbook_from_bytes` 传入 `original_filename`）。

### 1.2 Sheet 定位规则（综合 00表合集 多期归纳，非写死全名）

1. **归一化**（匹配前）：去掉常见书名号/括号、尾部 `!` `！`、尾部 `-能源局` / `--能源局`；全角 `＋` 视作 `+`。
2. **解析顺序**：对每一类逻辑表维护若干条规则，每条为 **须同时出现的子串（`all`）** 与 **不得出现的子串（`not_any`）**；按规则优先级依次扫描工作簿 **原始 Sheet 顺序**，**命中该规则的第一张表** 即选用。
3. **未命中**：该逻辑表视为缺失，对应指标 **空** 或子表 **无数据**。

**逻辑表与典型 Sheet 形态（实际以规则为准，下表为便于理解的代表名）**：

| 逻辑角色 | 典型命名形态 / 命中要点 |
|----------|-------------------------|
| 省份主表 | 含 **公共桩+各省+分交直流** 或 sheet 名即 **公共桩各省分交直流**；或 **公共桩保有量**（排除 **汇总**）；或 **充电桩+省份+建设**。**不**用「各省公共桩+地图」作主表（该表走备选 `header=1` 分支） |
| 省份备选 | **各省公共桩** 且 **地图** |
| 高速公路 | **高速公路**，排除 **运营商 / 充电站 / 各省分**；或 **11** 且 **高速** |
| 充电站分省宽表 | **充电站** 且 **各省** |
| 共享私桩分省 | **共享私桩**，排除 **公共+2**、**1公共** 等（避免命中「1公共+2共享私桩」） |
| 换电设施 / 换电电量 | **换电设施**；**换电** 且 **电量** 且排除 **月份** |
| 私桩分省 | **私桩** 且 **各省**，排除 **车企、增长、top10** |
| 各省充电电量 | **各省** 且 **充电电量**；或 **省+充电电量** 且排除 **运营商、同比、环比** |
| 运营商主序 | **运营商+新表**；若无 **运营商** 列再试 **运营商+排名**（排除 **新表**） |
| 公共+共享私桩 | **公共+2** 且 **共享** 等 |
| 运营商维度子表 | **运营商+公用桩/专用桩/直流桩/交流桩**（带排除避免串表）、**三相交流**、**运营商+充电功率**、**运营商+充电电量** |
| 充电站-运营商 | **充电站** 且 **运营商**，排除 **各省** |
| 私桩各车企 | **私桩** 且 **车企**，排除 **各省、增长** |
| 大功率 TOP5 制造商 | **250** 且 **制造**，或 **kW+制造**，或 **TOP5+制造** 等；**不会**仅凭 sheet 名 **top5** 命中（与「TOP5 运营商占比」类宽表区分） |

**表体内列名**：部分月份主表用 **合计(台)**、**交流桩数量(台)** 等，代码在读取后会 **别名到合计(个)**、**交流桩数量(个)** 等与解析逻辑一致的名字。

### 1.3 省份名称对齐

高速、充电站等子表可能出现 **「广东」**，主表为 **「广东省」**。实现通过 `_lookup_province_metric`：去 **省/市/自治区** 后缀及反向补全，在同一指标字典上查找，减少合并失败导致的空值。

## 二、输出工作簿结构

与桌面样表 **`00表标准化-系统输入-2602.xlsx`** 一致：

| Sheet | 内容说明 |
|-------|----------|
| **省份** | 列顺序见实现内 `PROVINCE_COLUMNS`；行来自主表各省 + 上表合并 |
| **运营商** | 列顺序见 `OPERATOR_COLUMNS`；主序来自 **1运营商新表** 按「数量」降序 |
| **城市** | 仅 **表头**（`城市`、`公共充电设施总量`），**无数据行** — 当前原始能源局簿无与标准一致的城市汇总 Sheet |
| **型号** | 来自 TOP5 制造商表 |
| **车企** | 来自 **5私桩各车企** |
| **换电站-运营商** | 仅 **表头**（`运营商`、`数量`），**无数据行** — 待原始簿提供可对齐子表后再接 |

| 标准列 | 数据来源要点 |

## 四、标准 00 表「运营商」列映射摘要

| 标准列 | 数据来源 |
|--------|----------|
| 运营商 | **1运营商新表**（或 1运营商排名） |
| 公共充电设施总量 | 上表第一个 **「数量」** 列 |
| 新增销量、星级场站数 | **空**（NA），原始汇总未接 |
| 共享私桩 | **1公共+2共享私桩**（表头须含运营商 + 共享私桩） |
| 公用充电桩 / 专用充电桩 / 直流桩 / 交流桩 / 三相交流桩 / 充电功率 / 充电电量（万度） | 对应 **1运营商-*** / **4三相交流** / **3运营商-充电功率** / **6运营商-充电电量** |
| 充电站 | **3充电站-运营商**（识别含「运营商」「站数量」等表头行） |

运营商名称在各子表中 **字符串精确匹配**；名称不一致时该列可能为 NA。

## 五、边界与限制

1. **城市、换电站-运营商**：当前输出 **空表占位**，上传至「由标准00表生成」后，依赖这些 Sheet 的七表子块 **无输入或为空**，与《由标准00表转化》对缺 Sheet 的约定一致。
2. **Sheet 更名**：若新命名仍能被现有 **关键词规则** 命中，一般 **无需改代码**；若出现全新形态，在 `resolve_raw_workbook_sheets` 中 **增一条规则** 即可（优先加 `all`/`not_any`，避免误匹配）。**表体结构大变**（列完全换义）时仍需改对应 `build_*` / `_series_*` 解析。
3. **省份行数**：以主表有效省行为准，与参考标准表总行数（如是否含「全国」等）可能差 1 行，以业务核对为准。
4. **文件名年月**：务必保留 `YYYYMM` 便于输出命名与后续多月 ZIP 管理。

## 六、界面与操作（Streamlit）

- **侧栏**：第四项 **「标准化数据产品-由原始表生成标准00表」**（与 `app.py` 中 `_NAV_OPTIONS` 一致）。
- **主区**：上传 xlsx → **「生成标准 00 表」** → **下载**；更换文件会清除上次生成的下载缓存键。
- 生成物可直接在第三项 **「标准化数据产品-由标准00表生成」** 中与其他月份 00 表一并上传转七表。

## 七、技术实现索引

- **`handlers/raw_to_standard00.py`**
  - `resolve_raw_workbook_sheets` / `load_sheet_plan` / `RawSheetPlan`：从工作簿 **全部 Sheet 名** 解析各逻辑表对应的真实表名。
  - `_norm_sheet_for_match`：Sheet 名归一化（去装饰后缀、标点等）。
  - `parse_yyyymm_from_filename`：文件名 → YYMM。
  - `build_province_dataframe` / `build_operator_dataframe` / `build_oem_dataframe` / `build_model_dataframe`：各 Sheet 数据框（均接受可选 `RawSheetPlan`，默认 `load_sheet_plan`）。
  - `build_standard00_workbook_bytes` / `build_standard00_workbook_from_bytes` / `build_standard00_workbook_from_uploaded`：写出 xlsx 字节流与建议文件名。
- **`app.py`**：` _render_raw_to_standard00_page()`，在 `view_mode == "标准化数据产品-由原始表生成标准00表"` 时调用。
