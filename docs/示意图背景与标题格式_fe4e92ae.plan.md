# 示意图背景与标题格式修改计划

## 一、示意图：再次缩小 + 置顶作背景 + 有字虚化 / 无字清晰

### 1.1 当前实现

- 示意图由 [`_show_asset_image`](d:\文件\充电代码工作\analysis\app.py) 渲染：PIL 缩放到原图高度的 **35%**，再以 `<img>` 显示，`opacity: 0.8`。
- 入库页：Banner（`st.markdown`）→ 标题「一键入库」→ `_show_asset_image("import-banner.png", ...)`。
- 标准化产品页：Banner → 标题「输出标准化产品」→ caption → `_show_asset_image("product-banner.png", ...)`。

### 1.2 目标

- **再次缩小**：高度改为当前的 **一半**，即原图高度的 **17.5%**（0.35 × 0.5）。
- **置顶作背景**：图作为「上面内容」的背景，即与 Banner + 标题在同一视觉区块内，图在底层。
- **有字虚化、无字清晰**：有文字的区域（上方）对图做虚化/遮罩，无文字的区域（图的下半部分）保持更清晰。

### 1.3 实现思路

- **布局**：不再单独画一个 `<img>`，而是把 **Banner + 标题 + 示意图** 合成 **一块 HTML**：
  - 外层一个 `div`（如 `position: relative`、固定或最小高度），**背景图**为缩放后的示意图（base64），`background-size` 控制为约一半高度（如 `auto 50%` 或按 17.5% 原图高度换算），`background-position: top`，置顶。
  - 在 **上方** 叠加一层渐变遮罩（`linear-gradient`）：从顶部半透明白/浅色到底部透明，使「有字」的上半部分被遮罩（视觉上更虚、更淡），下半部分无遮罩更清晰。
  - Banner 文案和页面标题放在该 `div` 内、`z-index` 更高，保证字在背景图之上。
- **缩放**：在 [`_show_asset_image`](d:\文件\充电代码工作\analysis\app.py) 或新函数中，将当前缩放比例从 `0.35` 改为 `0.175`（17.5%），仅用于生成背景图 base64。
- **复用**：入库页与标准化产品页都改为「一块带背景图的 HTML」：  
  入库页：同一块内包含「将清洗后的数据一键写入数据库」+「一键入库」+ 背景图 + 渐变；  
  标准化产品页：同一块内包含「生成与开放服务平台一致…」+「输出标准化产品」+ 背景图 + 渐变。  
  若文件不存在，该块仍显示现有 Banner + 标题，无背景图（或保留占位 div）。

### 1.4 涉及文件与改动点

- [app.py](d:\文件\充电代码工作\analysis\app.py)  
  - 新增或扩展函数：根据 `filename` 生成「带背景图 + 渐变 + 标题文案」的 HTML（可接受 banner 文案、主标题、是否显示 caption 等参数）；若 `assets/filename` 不存在则生成无背景图的 Banner+标题 HTML。  
  - 入库页：删掉当前独立的 Banner `st.markdown`、标题 `st.markdown` 和 `_show_asset_image` 调用，改为一次 `st.markdown(..., unsafe_allow_html=True)` 调用上述新 HTML。  
  - 标准化产品页：同样合并 Banner + 标题 + caption（如需）+ 背景图，一次输出。

## 二、标准化数据产品：标题格式与「数据库配置」一致

### 2.1 当前

- 入库页：使用 **`st.markdown("### 数据库配置")`**、**`st.markdown("### 入库方式")`**、**`st.markdown("### 执行入库")`** 作为区块标题。
- 标准化产品页：「数据来源」是 **`st.radio("数据来源", ...)`** 的 label；「上传清洗后的表格」是 **`st.file_uploader("上传清洗后的表格", ...)`** 的 label，没有单独的 ### 标题。

### 2.2 目标

- 「数据来源」和「上传清洗后的表格」的 **标题样式** 与入库页「数据库配置」一致，即使用 **`st.markdown("### ...")`** 作为区块标题。

### 2.3 具体改动（仅 app.py）

- 在 **标准化数据产品** 分支中：
  - 在 `st.radio` 前增加：`st.markdown("### 数据来源")`；  
    将 `st.radio` 的 label 改为简短说明或使用 `label_visibility="collapsed"`，避免与 ### 标题重复。
  - 在「导入文件」分支中，在 `st.file_uploader` 前增加：`st.markdown("### 上传清洗后的表格")`；  
    将 `st.file_uploader` 的 label 改为如「选择文件」或 `label_visibility="collapsed"`，避免与 ### 标题重复。
  - 若「从数据库选择表」下也有独立区块，可同样加 `st.markdown("### 选择表")` 或与「数据库配置」风格统一的标题，保持整页标题层级一致。

这样，「数据来源」「上传清洗后的表格」两个标题的格式就与入库模块的「数据库配置」一致（### 标题 + 下方控件）。

## 四、不确定点

- **虚化程度**：若仅用 `linear-gradient` 半透明遮罩仍觉得「有字处」不够虚，可在文字所在小容器上增加 `backdrop-filter: blur(4px)` 等，实现毛玻璃效果（需在 Streamlit 的 HTML 中测试兼容性）。
- **背景块高度**：背景图「置顶」且高度缩小后，整块 div 的最小高度可按「Banner + 标题 + 一小段留白」设定（如 `min-height: 180px`），避免块过矮导致背景图被裁切过猛。
