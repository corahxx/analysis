# 由标准00表转化 — 功能说明

本文档描述侧栏 **「标准化数据产品-由标准00表生成」** 能力：上传一张或多张 **00表**，勾选 **一个或多个统计月份**，**每个勾选月份各输出完整一套七表**（与线上一致的 Excel 结构），并打成一个 ZIP；在具备 **自然月上一期** 的 00 表时计算 **环比增量**、**环比增长率** 及 **市场占有率（占比）**。

**与现有功能关系**：不改变 **「标准化数据产品-由数据库生成」**（导入文件 / 从库选表 → 按需生成七产品）逻辑；本功能为 **侧栏独立入口**，**无需**先上传清洗明细表或选库表，互不共用 `product_cached_df`。

---

## 一、输入约定

### 1.1 文件命名与统计期

- 文件名需包含片段：**`00表标准化-系统输入-YYMM`**（不区分大小写），扩展名建议 **`.xlsx`**。
- **YYMM**：4 位数字，表示 **20YY 年 MM 月**（例：`2602` → 2026 年 2 月）。
- 示例：`00表标准化-系统输入-2602.xlsx`。
- 多文件：每个文件对应 **一个统计期**；**不允许**同一 YYMM 重复上传（若重复，实现中可报错或以后者为准，当前实现：**后者覆盖前者**并提示）。

### 1.2 工作簿 Sheet 与业务含义（与样表一致）

| 00表 Sheet | 用途 |
|------------|------|
| **省份** | 分省指标宽表：行=省份，列=各业务指标 → 映射到 **全国概况**（多 Sheet 分省表）、**省级数据**（每省一 Sheet 的维度行）、**车桩比**（按省） |
| **运营商** | 行=运营商，列=各运营商维度指标 → 映射到 **排行榜**（市场份额、设施销量、星级场站等）、**运营商概况**（多 Sheet） |
| **城市** | 行=城市 → **排行榜 · 城市榜 Top10**（列「城市」「公共充电设施总量」） |
| **型号** | 列：设备型号、装机量 → **型号榜** |
| **车企** | 列：车企名称、私桩安装量 → **车企私桩榜** |
| **换电站-运营商** | 行=运营商、列=数量（或兼容列名）→ **运营商概况 · 换电站** Sheet（**不再**从「运营商」Sheet 的换电站列取数） |

未出现的 Sheet 视为该块无输入：对应输出仍为 **标准表头**，数据列为空或零（见下文各产品）。

### 1.3 列名匹配规则

- 以 **表头文字** 为准，**去除首尾空格** 后做匹配。
- **别名**：允许「换电电量」与「换电电量（万度）」等带括号后缀与不带后缀的列互相兼容（实现内维护 **别名表**）。
- 若 00 表列名与下文章节映射表 **不一致** 或存在新业务列，以业务确认为准；当前实现已按桌面样表 `00表标准化-系统输入-2602.xlsx` 对齐。

## 二、输出目标（与现有七表一致）

### 2.1 单月与多月的 ZIP 结构

- **主流程（界面默认）**：一次下载 **一个 ZIP**，内含根文件夹 **`标准化数据产品_00表转化_{导出日期}`**（`导出日期` 为运行当日 `YYYYMMDD`）。其下 **每个勾选的统计期** 对应一个子文件夹 **`YYMM`**（如 `2601`、`2602`），每个子文件夹内 **固定 7 个文件**：
  - `01_车桩比.xlsx` … `07_运营商概况.xlsx`（命名与线上一键七表 ZIP 一致）。
- **示例**：勾选 2026 年 1 月、2 月 → ZIP 内为 `…/2601/*.xlsx` 与 `…/2602/*.xlsx` 两套。
- **兼容**：代码仍保留单期打包函数 `build_standard00_zip_bytes`（根目录为 `标准化数据产品_00表转化_{YYMM}_{导出日期}`），供脚本或旧习惯调用；页面端以 **多期合一 ZIP** 为准。

生成内容与 `handlers/product_bundle.py` 一键 ZIP 中七类文件 **同结构**（Sheet 名、列名、顺序一致）。

| 序号 | 产品 | 说明 |
|------|------|------|
| 01 | 车桩比 | 列：省份、新能源车保有量、公共充电设施数量、车桩比、评价 |
| 02 | 高速公路 | **业务模板双 Sheet**，与需求表 **`模板_高速公路_样例.xlsx`** 一致（路径优先：`C:\Users\HONOR\Desktop\充电标准七张表输出\充电系统标准数据产品数据需求表\模板_高速公路_样例.xlsx`，否则使用项目内 `assets/模板_高速公路_样例.xlsx`）。Sheet：`高速公路沿线已建设及预留建设充电停车位服务区`、`高速公路沿线已建设充电停车位总数`；表头「省份」「数量」，数据由 00 表「省份」同名列填入。一键七表 ZIP 同逻辑。 |
| 03 | 功率段分布 | 00表 **无功率分档明细**：按 **省份 Sheet 中出现的省份** 各生成一 Sheet，五档 **数量=0、占比=0.00%、环比空**（结构对齐线上） |
| 04 | 排行榜 | 六 Sheet：市场份额榜、设施销量榜、城市榜Top10、星级场站榜、型号榜、车企私桩榜 |
| 05 | 全国概况 | 十一 Sheet，列：省份、数量、全国占比、环比增速 |
| 06 | 省级数据 | 每省一 Sheet，列：数据维度、数值、环比变化 |
| 07 | 运营商概况 | 十一 Sheet，列：运营商、数值、环比变化、环比增速 |

## 三、核心计算口径

### 3.1 市场占有率 / 全国占比 / 占比（分母一致）

对某一 **Sheet**、某一 **数值指标列**（如运营商的「公共充电设施总量」、城市的「公共充电设施总量」、车企的「私桩安装量」、星级榜的「星级场站数」等）：

- **占比** = 该行数值 ÷ **该 Sheet 内该指标列可求和的所有行数值之和**。
- **显示格式**：百分比字符串 **固定保留小数点后两位**（如 `89.73%`、`3.05%`）。
- **合计约束**：为避免逐项四舍五入导致全表占比之和偏离 100%，实现采用 **最大余额法** 在万分之一精度上分配后输出，保证 **同一批行的占比数值之和恰为 100.00%**（与手工按总值验算一致）。
- **全国概况** 中「全国占比」列：含义为 **各省该指标占该 Sheet 合计的比例**，口径与上式一致（分母为各省数量之和）。

### 3.2 环比增量

- 针对 **同一维度键**（同一省份、同一运营商、同一城市、同一型号、同一车企等）：
  - **环比增量** = **本期（目标统计期）** 该指标值 − **上期（自然月上一期）** 该指标值。
- **上期数据**来源：用户上传文件中 **是否存在** 上一自然月的 YYMM。
  - 若 **未上传** 上月文件：环比增量 **空**（导出为单元格空，不写 0）。
  - 若上月已上传但某键在上月不存在：该键环比增量 **空**。

### 3.3 环比增长率

- 在同时有 **本期值** `curr` 与 **上期值** `prev` 时：
  - 若 `prev != 0`：**环比增长率** = `(curr - prev) / prev * 100`，格式化为 **一位小数 + `%`**（如 `5.2%`）。
  - 若 `prev == 0` 且 `curr != 0`：记为 **`—`**（或空，实现统一为 **`—`** 以与全国概况旧占位一致）。
  - 若 `prev == 0` 且 `curr == 0`：**空** 或 **`0.0%`**（实现：**`0.0%`**）。
- 列名与线上一致处：**环比增速**、**全国概况** 中为「环比增速」；运营商表中为「环比增速」；省级表中「环比变化」列存放 **增量（数值或带符号字符串）**，与线上一列命名保持兼容。

### 3.4 勾选生成月份与环比条件

- 解析成功的 **所有统计期** 在 **侧栏「标准化数据产品-由标准00表生成」** 页中以 **复选框** 展示（文案含「YYYY年MM月（YYMM）」）；默认 **全部勾选**；用户 **勾选哪几个月，就为那几个月各生成一套七表**。
- 各月 **本期** 数据仅来自 **该月对应的 00 表**；**环比** 仅在与 **自然月上一期** 的 00 表 **均已上传** 时，按 3.2、3.3 填充。
- **若某勾选月份没有上传「上个月」的标准 00 表**：该月那一套七表中，凡 **环比增量、环比增速、环比变化** 等环比类列 **一律留空**（或按实现约定为 `—` 的仍为 `—`），**不推算、不补零**。
- ZIP 内多个月份子目录按 **时间升序** 写入。

## 四、00表 → 七表 字段映射（实现基准）

### 4.1 省份 Sheet → 全国概况（分省「数量」）

| 全国概况 Sheet | 00表「省份」Sheet 列名（优先左列） |
|----------------|-------------------------------------|
| 充电站 | 充电站 |
| 公共充电桩 | 公共充电桩 |
| 交流桩 | 交流桩 |
| 直流桩 | 直流桩 |
| 充电电量 | 充电电量 |
| 共享私桩 | 共享私桩 |
| 换电站 | 换电站 |
| 换电电量 | 换电电量（万度）、换电电量 |
| 私桩及个人充电设施 | 私桩及个人充电设施 |
| 随车配建 | 随车配建 |
| 交直流桩 | 交直流桩数量 |

列缺失时：该 Sheet 仅 **表头**、无数据行。

### 4.2 省份 Sheet → 省级数据（每省 Sheet 内「数据维度」行）

维度行顺序固定为（与 `province_handler.PROVINCE_DIMENSION_ROWS` 一致）：

`充电站` · `公共充电桩` · `交流桩` · `直流桩` · `充电电量` · `共享私桩` · `换电站` · `换电电量` · `私桩/个人充电设施` · `随车配建` · `交直流桩`

| 数据维度 | 00表列 |
|----------|--------|
| 充电站 | 充电站 |
| 公共充电桩 | 公共充电桩 |
| 交流桩 | 交流桩 |
| 直流桩 | 直流桩 |
| 充电电量 | 充电电量 |
| 共享私桩 | 共享私桩 |
| 换电站 | 换电站 |
| 换电电量 | 换电电量（万度）、换电电量 |
| 私桩/个人充电设施 | 私桩及个人充电设施 |
| 随车配建 | 随车配建 |
| 交直流桩 | 交直流桩数量 |

- **数值**：取该省行对应列（数值化失败为空）。
- **环比变化**：该单元格 **环比增量**（字符串形式，可为负数；无上月为空）。

### 4.3 省份 Sheet → 车桩比

| 输出列 | 来源 |
|--------|------|
| 省份 | 省份 |
| 新能源车保有量 | 新能源车保有量 |
| 公共充电设施数量 | **公共充电设施数量**；若缺列则用 **公共充电桩** |
| 车桩比 | 若保有量、设施数量均为有效正数：保有量÷设施数量，保留 **4 位小数**；否则空 |
| 评价 | 空 |

### 4.4 运营商 Sheet → 排行榜

- **市场份额榜**：运营商、公共充电设施总量（列名一致）、市场份额（按 3.1 计算）、环比增速（按 3.3，对比上月同运营商列「公共充电设施总量」）。
- **设施销量榜**：运营商列表与 00 表一致；**新增销量** 取列「新增销量」；**环比增量**、**环比增速** 对「新增销量」按 3.2、3.3。
- **星级场站榜**：运营商、星级场站数（列「星级场站数」）、占比（对星级场站数做 3.1）、五星级场站数（00表无此列 → **整列空**）。

### 4.5 运营商 Sheet → 运营商概况

Sheet 顺序与线上一致。列名 **与 00 表「运营商」Sheet 一致** 时直接映射为「数值」列：

公共充电设施 ← 公共充电设施总量；共享私桩、公用充电桩、专用充电桩、直流桩、交流桩、三相交流桩、充电功率、充电站；**充电电量** ← 充电电量（万度）或 充电电量。

**换电站** Sheet：**单独** 读取 00 表 **`换电站-运营商`** Sheet：「运营商」+ 数值列优先 **「数量」**（次选「换电站」）。环比与上期同 Sheet、同运营商对齐。

「环比变化」「环比增速」对应该 Sheet 数值列做 3.2、3.3。无数据 Sheet 仅表头。

### 4.6 城市 / 型号 / 车企 Sheet

- **城市榜 Top10**：数据 **仅来自** 00 表 **「城市」** Sheet；列「城市」「公共充电设施总量」。城市名列按 `str(单元格).strip()` 展示，空则跳过；按总量降序取前 10；**全国占比** 的分母为 **本 Sheet 内这 10 城「公共充电设施总量」之和**（十城占比合计 100.00%，算法见 3.1）；环比增速按 3.3（依赖上月「城市」Sheet）。
- **型号榜**：设备型号、装机量（无数值列则空表）。
- **车企私桩榜**：车企名称、私桩安装量、占比（3.1）、环比增速（3.3）。

### 4.7 省份 Sheet → 高速公路（02）

- 输出文件结构由 **`模板_高速公路_样例.xlsx`** 决定；加载顺序见 `handlers.highway_template.HIGHWAY_TEMPLATE_DESKTOP`，其次 `assets/模板_高速公路_样例.xlsx`。若均缺失则回退为简短说明 Sheet。
- 两个 Sheet 名与「省份」表中下列列名一致，逐省填入 **数量**：
  - `高速公路沿线已建设及预留建设充电停车位服务区`
  - `高速公路沿线已建设充电停车位总数`

## 五、边界与待业务确认项

1. **功率段分布**：00表未提供分档明细时，仅保证 **文件结构** 与线上一致，**不猜测** 分档数据。
2. **高速公路模板**：优先读取桌面需求表路径（见 `highway_template.HIGHWAY_TEMPLATE_DESKTOP`）；无则读 `assets/模板_高速公路_样例.xlsx`。
3. **文件名**：若不匹配 `00表标准化-系统输入-YYMM`，该文件 **跳过** 并在界面 **警告**。
4. **新能源车保有量**：样表中多为占位 `1`；车桩比仍按公式计算，是否改口径由业务决定。
5. **列名扩展**：新增指标列时，需在映射表中增补或产品确认后再开发。

## 六、界面与操作（Streamlit）

- **侧栏导航**：与「入库」「标准化数据产品-由数据库生成」并列的第三项 **「标准化数据产品-由标准00表生成」**；进入后 **仅** 处理 00 表，**不经过** 数据库生成页的「数据来源」与 `st.stop()`，无需先导入清洗表。
- **主区组件**：
  1. `st.file_uploader`（`accept_multiple_files=True`，类型 xlsx/xls）；
  2. 解析成功后展示 **复选框网格**（按统计期一行多列排布）：每个 YYMM 一个 `checkbox`，**默认勾选**；仅导出仍勾选的月份；
  3. 按钮 **「生成七表 ZIP（由 00 表）」**：至少勾选一个月份；
  4. `st.download_button` 下载（文件名含所选 YYMM 片段）。
- 上传文件列表变化时，会重置各月份复选框的 session 键，默认重新 **全选**。

## 七、技术实现索引

- `handlers/highway_template.py`：`build_highway_workbook_bytes(province_df)`，按 `resolve_highway_template_path()`（桌面模板优先）加载 xlsx 并按省份填数；`product_bundle` 无省份数据时传 `None` 得空表模板。
- `handlers/standard00_transform.py`  
  - `format_pct_share_strings_two_dp`：占比两位小数 + 最大余额法合计 100.00%。  
  - `build_standard00_multi_month_zip_bytes`：多期合一 ZIP。  
  - `_write_seven_standard00_into_zip`：单月七表写入 Zip。  
  - `build_standard00_zip_bytes`：单期 ZIP（兼容）。  
  - `_swap_station_operator_table`：「换电站-运营商」→ 表7「换电站」。
- 入口：`app.py` 中 **`_render_standard00_transform_page()`**，由侧栏 `view_mode == "标准化数据产品-由标准00表生成"` 调用；侧栏选项列表见 `_NAV_OPTIONS`（旧文案「由标准00表转化」「标准化数据产品」会在会话中自动迁移为新名称）。
